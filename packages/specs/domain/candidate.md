# Domain: Candidate

## Fields (slice 1)

- `id`
- `name`
- `role`
- `technologies`
- `expectedSalary`
- `available`
- `status` (`Active|Inactive|Pending`)
- optional `email`, `phone`
- timestamps

## Behaviors

- CRUD in admin candidates page.
- Filter/search/sort list query endpoint.
- Row actions for edit/remove and invite-link generation.
